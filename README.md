# Black Russia Complaints Bot

Telegram-бот для автоматической подачи жалоб на форум Black Russia (XenForo).
Логинится сам, обходит DDoS-Guard JS-челлендж, держит пул форумных аккаунтов
с кулдауном, проверяет статусы поданных жалоб и шлёт уведомления, когда
тему приняли, отклонили или закрыли.

## Возможности

- **Логин по паролю** с поддержкой 2FA (email-код), сессия сохраняется в БД
- **Шифрование** сохранённого пароля (Fernet, ключ в `SECRET_KEY`)
- **Мульти-аккаунты**: пул форумных аккаунтов админа, ротация, кулдаун 180с
- **Автообнаружение** структуры форума: серверы и категории жалоб (`/sync`)
- **Шаблоны жалоб**: DM / DB / Оск. родителей / Чит-ПО + свои
- **Валидация ввода**: ник латиницей, дата, мат-фильтр, запрет VK/OK как пруфов
- **Автозагрузка скриншотов** на imgbb (если задан `IMGBB_API_KEY`)
- **Очередь жалоб**: отложенная публикация, фоновый процессор с retry
- **Статусы и пуш-уведомления**: фоновый чекер каждые 5 минут, пушит когда
  префикс темы поменялся (Принято / Отклонено / Закрыто). В уведомление
  вкладывается текст последнего ответа админа из темы.
- **Перебор аккаунтов** при проверке статусов: жалобу видит автор и админы,
  поэтому если у активного аккаунта нет прав — пробуется весь пул.
- **Удаление и редактирование** жалоб на форуме (от имени аккаунта-автора)
- **Черновики**: автосохранение FSM, продолжение по `/draft`
- **Баг-репорты** от пользователей с inline-ответом админа
- **Бэкап БД** в приватный Telegram-канал (с дебаунсом 30с) и автовосстановление
  при старте — для хостингов без постоянного диска
- **Режим обслуживания** (`/maintenance on/off`) — закрывает доступ всем
  кроме админов
- **Антиспам** middleware: rate-limit, soft-ban, защита от длинных сообщений
- **Статистика** с графиками matplotlib (`/stats`) и **рассылка** (`/broadcast`)

## Требования

- Python 3.11+
- Аккаунт на forum.blackrussia.online с правами публикации в разделах жалоб
- Telegram-бот ([@BotFather](https://t.me/BotFather))

## Конфигурация

Скопируйте пример и заполните своими данными:

```bash
cp .env.example .env     # Linux/macOS
copy .env.example .env   # Windows
```

Откройте `.env` и подставьте:

| Переменная           | Назначение |
|----------------------|------------|
| `BOT_TOKEN`          | Токен от @BotFather |
| `ADMIN_IDS`          | Telegram ID админов через запятую (узнать у @userinfobot) |
| `FORUM_URL`          | URL форума (по умолчанию `https://forum.blackrussia.online`) |
| `USER_AGENT`         | UA для прохождения DDoS-Guard. По умолчанию свежий Chrome |
| `IMGBB_API_KEY`      | Ключ для автозагрузки скриншотов на imgbb.com (опционально) |
| `STORAGE_CHANNEL_ID` | ID приватного Telegram-канала для бэкапа БД (опционально) |
| `SECRET_KEY`         | Fernet-ключ для шифрования паролей (опционально) |
| `LOG_TO_ADMIN`       | `1` — слать ERROR/CRITICAL первому админу в личку |
| `ERROR_LOG_CHAT_ID`  | Альтернатива: chat_id куда слать ошибки |

Сгенерировать `SECRET_KEY`:

```bash
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

## Запуск

```bash
python main.py            # точка входа для большинства хостингов
# или
python -m src.bot         # эквивалентно
```

Бот слушает long-polling. Логи пишутся в консоль и в `logs/bot.log` (ротация 2МБ × 5).

## Первый запуск

1. Отправьте боту `/start`.
2. Нажмите `🔐 Войти по паролю` (только админ). Введите логин/пароль.
3. Если включена 2FA — придёт код на почту, введите в чат.
4. Бот сохранит сессию в БД, активирует её и предложит сохранить пароль.
5. Запустите `🔄 Синхронизировать форум` — соберутся серверы и подразделы.
6. Готово. Любой пользователь может нажать `📝 Подать жалобу`.

```
├── maintenance.py      # переключатель режима обслуживания
├── middleware.py       # rate-limit, soft-ban, maintenance gate
├── queue_processor.py  # фоновый воркер очереди жалоб
├── status_monitor.py   # фоновая проверка статусов жалоб
├── storage_backup.py   # бэкап SQLite в Telegram-канал
├── uploader.py         # загрузка картинок на imgbb
├── validation.py       # валидаторы полей + мат-фильтр
├── charts.py           # графики статистики (matplotlib)
├── forum/
│   ├── xenforo.py      # авторизация, DDoS-Guard, публикация, парсинг
│   └── templates.py    # шаблоны тел жалоб + правила оформления
└── handlers/
    ├── common.py       # /start, /login, /sync, /accounts, /me, /cancel
    ├── complaint.py    # подача жалобы, история, шаблоны, редактирование
    ├── bugreport.py    # баг-репорты от пользователей и ответы админа
    └── admin.py        # /stats, /broadcast, /check, /queue, /maintenance
```

## Деплой на TelegramBotHost / VPS

### TelegramBotHost

1. В настройках хостинга задайте `python main.py` как стартовую команду.
2. Прокиньте все `.env`-переменные через панель управления.
3. Чтобы БД и куки переживали рестарты, поставьте:
   - `DB_PATH=/app/shared/bot_database.db`
   - `COOKIES_PATH=/app/shared/cookies.json`
4. Дополнительно настройте `STORAGE_CHANNEL_ID` — бот будет дублировать БД
   в приватный канал, и при старте автоматически восстанавливать её.

### VPS (systemd)

Минимальный юнит `/etc/systemd/system/br-bot.service`:

```ini
[Unit]
Description=Black Russia Complaints Bot
After=network-online.target

[Service]
Type=simple
User=brbot
WorkingDirectory=/opt/br-bot
EnvironmentFile=/opt/br-bot/.env
ExecStart=/opt/br-bot/.venv/bin/python main.py
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now br-bot
sudo journalctl -u br-bot -f
```

## Безопасность

- `.env`, `cookies.json`, `bot_database.db` и `logs/` в `.gitignore` — не коммитятся.
- Сообщение с паролем при логине автоматически удаляется из чата (с retry).
- Пароль не сохраняется в логах. В БД хранится только Fernet-зашифрованным.
- HTML-разметка с эскейпом всех динамических строк через `html.escape()` —
  защита от инъекций в заголовки тем и комментариях.
- Все SQL — параметризованные. SQL-инъекции исключены.
- BEGIN IMMEDIATE + UPDATE...RETURNING для атомарного выбора аккаунта без race.
