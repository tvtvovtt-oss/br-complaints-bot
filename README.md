# Black Russia Complaints Bot

Telegram-бот для автоматической подачи жалоб на форум Black Russia (XenForo).
Логинится на форум сам (включая 2FA через email), решает DDoS-Guard JS-челлендж,
автоматически выбирает свободный форумный аккаунт из пула, проверяет статусы
поданных жалоб и шлёт уведомления когда тему приняли или отклонили.

## Возможности

- **Логин по паролю** с поддержкой 2FA (email-код), куки сохраняются в БД
- **Мульти-аккаунты**: пул форумных аккаунтов админа, ротация, кулдаун 180с
- **Автообнаружение** структуры форума: 91 сервер + 4 категории жалоб у каждого
- **Шаблоны жалоб**: DM, DB, nRP Drive, Оск. родителей, Чит-ПО + свои
- **Валидация** ввода: ник латиницей, дата, мат-фильтр, запрет VK/OK как пруфов
- **Статусы и уведомления**: фоновый чекер каждые 5 минут с пушами в Telegram
- **Удаление** жалоб из истории (тема на форуме остаётся)
- **Публичный режим**: обычные пользователи только подают жалобы; админы
  управляют пулом аккаунтов и синхронизацией

## Требования

- Python 3.11+
- Аккаунт на forum.blackrussia.online с правами публикации в разделах жалоб
- Telegram-бот ([@BotFather](https://t.me/BotFather))

## Установка

```bash
git clone <ваш-репозиторий>
cd <папка>
python -m venv .venv

# Linux/macOS
source .venv/bin/activate

# Windows (PowerShell)
.venv\Scripts\Activate.ps1

pip install -r requirements.txt
```

## Конфигурация

Скопируйте пример и заполните своими данными:

```bash
cp .env.example .env   # Linux/macOS
copy .env.example .env # Windows
```

Откройте `.env` и подставьте:

- `BOT_TOKEN` — токен от @BotFather
- `ADMIN_IDS` — ваш Telegram ID (узнать можно у @userinfobot)
- `USER_AGENT` — оставьте дефолтный или поставьте свой Chrome/Firefox

## Запуск

```bash
python -m src.bot
```

Бот слушает long-polling. Логи пишутся в консоль и в `logs/bot.log` с ротацией.

## Первый запуск

1. Отправьте боту `/start`.
2. Нажмите `🔐 Войти по паролю` (только для админа). Введите логин/пароль.
3. Если включена 2FA — придёт код на почту, введите его в чат.
4. Бот сохранит сессию в БД, сделает её активной.
5. Запустите `🔄 Синхронизировать форум` — соберутся серверы и подразделы жалоб.
6. Готово. Теперь любой пользователь может нажать `📝 Подать жалобу`.

## Структура

```
src/
├── bot.py              # точка входа aiogram
├── config.py           # парсинг .env
├── database.py         # SQLite (aiosqlite)
├── effects.py          # ID анимированных эффектов Telegram
├── logger.py           # русские логи + ротация
├── status_monitor.py   # фоновая проверка статусов жалоб
├── validation.py       # валидаторы полей + мат-фильтр
├── forum/
│   ├── xenforo.py      # авторизация, публикация, парсинг форума
│   └── templates.py    # шаблоны тел жалоб + правила оформления
└── handlers/
    ├── common.py       # /start, /login, /sync, аккаунты, статус
    └── complaint.py    # сценарий подачи жалобы, история, шаблоны
```

## Деплой на сервер (Linux)

Минимальный systemd-юнит `/etc/systemd/system/br-bot.service`:

```ini
[Unit]
Description=Black Russia Complaints Bot
After=network-online.target

[Service]
Type=simple
User=brbot
WorkingDirectory=/opt/br-bot
Environment="PATH=/opt/br-bot/.venv/bin"
ExecStart=/opt/br-bot/.venv/bin/python -m src.bot
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now br-bot
sudo journalctl -u br-bot -f
```

## Команды бота

**Все пользователи:**
- `/start` — приветствие и главное меню
- `/help` — справка
- `📝 Подать жалобу` (`/new_complaint`) — пошаговый сценарий
- `📜 Мои жалобы` — история со статусами и удалением
- `📋 Мои шаблоны` (`/templates`) — управление личными шаблонами

**Только админ:**
- `🔐 Войти по паролю` (`/login`) — добавить форумный аккаунт
- `👥 Аккаунты` (`/accounts`) — переключение и удаление аккаунтов
- `🔄 Синхронизировать форум` (`/sync`) — пересобрать список серверов
- `🔍 Проверить статус форума` — проверка всех аккаунтов
- Загрузка `cookies.json` файлом в чат

## Безопасность

- `.env` и `cookies.json` в `.gitignore` — не коммитятся.
- Сообщение с паролем при логине автоматически удаляется из чата.
- Пароль не сохраняется ни в БД, ни в логах. В логи пишется только логин.
- В сообщении о неуспешном логине показываются причины без секретов.

## Лицензия

Проект только для образовательных целей. Использование в обход правил
форума — на свой страх и риск.
